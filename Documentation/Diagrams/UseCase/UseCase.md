 # 2 Варианты использования <a name="2"/>
 
 ## 2.1 Получение информации о состоянии баланса <a name="2.1"/>

 **Описание.** Вариант использования "Получение информации о состоянии баланса" дает пользователю возможность просматривать состояние баланса, затраченные и имеющиеся средства, состояние копилки, а также долги.

 **Основной поток.**
 1. Приложение отображает пользователю главный экран.
 2. Приложение обращается в хранилище и обновляет соответсвутющие поля.
 3. Главный экран отображает пользователю информацию о балансе.
 4. Вариант использования завершается.
 
 ## 2.2 Добавление средств к балансу <a name="2.2"/>

 **Описание.** Вариант использования "Добавление средств к балансу" позволяет пользователю добавить некоторую сумму уже к имеющимся средстам.

 **Основной поток.**
 1. Приложение отображает пользователю главный экран.
 2. Пользователь нажимает на кнопку "Установка/Изменение баланса".
 3. Приложение отображает пользователю экран внесения изменений в баланс.
 4. Пользователь вводит сумму и нажимает на кнопку "Добавить".
 5. Введенная сумма добавляется к текущему балансу, если же веденное значение не является числом, то выполняется альтернативный поток А1.
 6. Приложение обновляет состояние базы данных.
 7. Приложение отображает главный экран с обновленным балансом.
 8. Вариант использования завершается.
 
 **Альтернативный поток А1.**
 1. Приложение отображает сообщение об ошибке ввода данных.
 2. Возврат к п.4 основного потока.

 ## 2.4 Разделение затрат по категориям <a name="2.4"/>

 **Описание.** Вариант использования "Разделение затрат по категориям" позволяет пользователю распределять затраченные средства по категориям: еда, одежда, учеба, развлечения и т.д.

 **Основной поток.**
 1. Приложение отображает пользователю главный экран.
 2. Пользователь нажимает на кнопку "ToolBar".
 3. Приложение отображает пользователю ToolBar.
 4. Пользователь нажимает на кнопку "Категории".
 5. Приложение отображает экран категорий с распределенными затратами.
 6. Пользователь нажимает на кнопку "Добавить запись".
 7. Приложение отображает экран добавления затрат в определенную категорию.
 8. Пользователь вводит данные или нажимает кнопку "Назад". В последнем случае выполняется альтернативный поток А3.
 9. Затраты добавлются в выбранную категорию, если же параметры не были выбраны верно, то выполняется альтернативный поток А4.
 10. Приложение обновляет состояние базы данных.
 11. Приложение отображает экран категорий с обновленными данными.
 12. Вариант использования завершается.

 **Альтернативный поток А3.**
 1. Приложение сбрасывает окно ввода данных.
 2. Возврат к п.5 основного потока.

 **Альтернативный поток А4.**
 1. Приложение отображает сообщение об ошибке ввода данных.
 2. Возврат к п.7 основного потока.
